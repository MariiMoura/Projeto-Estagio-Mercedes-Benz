Este projeto consiste em uma automação desenvolvida em Python para otimizar a análise de Requisições de Compra (PRs) do tipo J dentro do SAP, eliminando a necessidade de verificações manuais repetitivas. Utilizando SAP GUI Scripting, a aplicação acessa automaticamente a transação ME53N, abre cada requisição de compra, navega até a ordem vinculada e verifica se existe um Elemento PEP, responsável por identificar se a despesa está relacionada a um investimento.
A solução realiza a leitura de uma planilha Excel contendo as PRs, processa apenas as requisições do tipo J, registra automaticamente o resultado da análise na coluna "Comentários" e gera um novo arquivo com as informações atualizadas. Além disso, o sistema possui uma interface gráfica com barra de progresso, estimativa de tempo restante, opção de cancelamento e salvamento automático periódico, reduzindo o risco de perda de dados durante a execução.
O projeto foi desenvolvido com foco em confiabilidade e recuperação de erros, incluindo mecanismos para reconectar e reposicionar corretamente a navegação no SAP, tratamento de exceções e validações que permitem a continuidade do processamento mesmo diante de falhas pontuais.
Com essa automação, o processo torna-se significativamente mais rápido, preciso e padronizado, reduzindo o esforço operacional da equipe, minimizando erros humanos e aumentando a produtividade na análise de requisições de compra.
